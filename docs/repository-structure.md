## How the workspaces depend on each other

```
                ┌──────────────────┐
                │   shared (TS)    │  openapi.yaml + TS types
                └────────┬─────────┘
                         │ file:../shared
              ┌──────────┴──────────┐
              ▼                     ▼
   ┌──────────────────┐   ┌──────────────────┐
   │  frontend-web    │   │  frontend-mobile │
   └────────┬─────────┘   └────────┬─────────┘
            │ HTTP                  │ HTTP
            └──────────┬────────────┘
                       ▼
            ┌────────────────────┐
            │      backend       │  reads shared/openapi.yaml at build time
            └─────────┬──────────┘    (contract test via Schemathesis)
                      ▼
        ┌────────────────────────────┐
        │  Postgres / Redis / Meili  │
        └────────────────────────────┘
```

- `shared` has **no runtime dependencies on other workspaces**.
- Both frontends import shared via `file:../shared` so changes to the contract
  appear immediately during development.
- Backend doesn't import shared at runtime, but its CI runs Schemathesis
  against `shared/openapi.yaml` to enforce that the implementation matches the
  contract.

## Conventions per workspace

| Workspace | Lint | Format | Test | Type-check |
| --- | --- | --- | --- | --- |
| backend | `ruff check` | `ruff format` | `pytest` | `mypy` |
| frontend-web | `eslint` | `prettier` | `vitest` + `cypress` | `tsc --noEmit` |
| frontend-mobile | `eslint` | `prettier` | `jest` | `tsc --noEmit` |
| shared | (n/a) | `prettier` | (none) | `tsc --noEmit` |

CI enforces lint, type-check, and unit tests on every PR. Cypress smoke runs
on every PR; the full Cypress suite runs nightly.

## Why a monorepo?

- **One PR can update the contract + backend + frontend** atomically.
- **Shared types** are checked at compile time across workspaces.
- **One CI configuration** style; per-workspace `paths` filters keep runs fast.
- **One issue tracker, one release cadence.**

The trade-off — you can't deploy services from independent repos — is fine for
a marketplace where backend/frontend always ship together.
